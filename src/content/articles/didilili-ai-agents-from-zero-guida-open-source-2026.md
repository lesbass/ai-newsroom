## Perché interessa

**1. La copertura di stack coincide con la pila AI agent 2026 adottata in produzione.** Una guida unica che tocchi **MCP + LangGraph + RAG + Skills + fine-tuning** sotto MIT, in un solo syllabus, è rara. Le risorse aperte su questi temi sono di solito sparse in 5–10 repository distinti (es. [LangChain Python docs](https://python.langchain.com/docs/introduction/), [LangGraph docs](https://langchain-ai.github.io/langgraph/), [MCP spec](https://modelcontextprotocol.io/introduction), tutorial RAGAS separati, tutorial Llama-Factory separati). Avere i capitoli 1–27 nello stesso posto, con uno stesso stile e un unico sistema di glossario (`全书术语表.md`, 62 KB), abbassa il costo di orientamento per chi deve costruire un'architettura multi-componente.

**2. I due progetti completi sono il valore reale, non la teoria.** `电商问数` (NL2SQL su MySQL+LangGraph+Qdrant+Elasticsearch) e `深度研搜` (DeepAgents multi-agent con WebSocket live progress) sono eseguibili: copiarli, far girare il `docker compose` locale, leggere il codice Python, e si ottiene una pipeline reale RAG + multi-agent + REST + WebSocket. Per un team italiano che vuole vedere «come si mette insieme questo tipo di sistema in produzione» sono un case study raro open source, e le date di completamento (3 maggio e 17 maggio 2026) li rendono attuali.

**3. La barriera linguistica va dichiarata apertamente.** Il README, la sidebar, le 27 `.md` e la interview question bank sono **in cinese** (zh-cn, con nomi di file in caratteri cinesi: `9-LangChain概述与架构.md`, `20-MCP模型上下文协议.md`, `AI智能体与大模型应用开发面试题库.md`). I sample di codice sono in **Python**, leggibili da chiunque conosca il linguaggio. Un articolo che promette «è in inglese» sarebbe sbagliato al 13 giugno 2026: il branch `main` è cinese. Un articolo che dice «il codice si legge, la prosa no» è onesto e consente al lettore italiano di decidere se vale il proprio tempo di traduzione (o se leggere solo i sorgenti Python è sufficiente).

**4. Il pattern «tutorial lungo + archivio di domande da colloquio + progetti reali» è un formato industriale cinese.** L'[interview question bank `AI智能体与大模型应用开发面试题库.md`](https://github.com/didilili/ai-agents-from-zero/blob/main/AI%E6%99%BA%E8%83%BD%E4%BD%93%E4%B8%8E%E5%A4%A7%E6%A8%A1%E5%9E%8B%E5%BA%94%E7%94%A8%E5%BC%80%E5%8F%91%E9%9D%A2%E8%AF%95%E9%A2%98%E5%BA%93.md) è allineato al profilo di job description «AI 智能体 / 大模型应用开发工程师» (AI Agent / LLM Application Engineer), una figura professionale standard in Cina nel 2026. ## Rischi e caveat

1. **Il README è in cinese, non in inglese.** Verificato il 13 giugno 2026 sul branch `main` via REST API e via rendering del README grezzo. Una traduzione EN ufficiale o non non è ancora parte del branch canonico.
2. **Il syllabus è allineato al profilo «AI 智能体 / 大模型应用开发工程师» del mercato del lavoro cinese 2026.** Non è automaticamente trasferibile al mercato italiano o EU. I titoli delle posizioni, le aspettative sui colloqui e i tagli tematici riflettono quel mercato, non quello europeo.
3. **«Completo» significa «27 capitoli + 2 progetti», non «onnicomprensivo».** Non è la guida definitiva per l'AI agent: il README stesso la definisce una «速成指南» (guida rapida). È un punto di partenza sistematico, non un riferimento esaustivo.
4. **La licenza MIT permette fork e ridistribuzione con attribuzione**, ma l'articolo non cita fork di terzi a meno che non siano fonti primarie verificabili. Il sub-repo `deepsearch-agents` non ha un file `LICENSE` visibile nei metadata GitHub al 13 giugno 2026 (mentre `shopkeeper-agent` ha MIT); per il sub-repo vale il principio di non fare affermazioni di licenza non verificate.
5. **I numeri di stelle / fork / commit crescono nel tempo.** 1.914 stelle e 254 fork sono osservati il 13 giugno 2026; non è un'istantanea permanente. Le date di osservazione vanno citate per ogni numero.
6. **Non è una libreria installabile.** Il repository non ha rilasci PyPI né npm. È un content + code repository, da clonare o leggere, non da `pip install`.
7. **Il repository ha una sponsorizzazione dichiarata.** [UCloud AI cloud (优云智算)](https://www.compshare.cn/coding-plan) è citata esplicitamente nel README come sponsor. È una disclosure onesta, non un bias editoriale, e l'articolo la segnala come trasparenza — non è un endorsement del servizio.
8. **L'articolo non confronta la guida con LangChain tutorials ufficiali, LangGraph tutorials, DeepAgents tutorials, MCP spec, RAGAS, o corsi a pagamento in termini di «qualità» o «completezza».** Quei confronti non sono fattuali senza metriche misurabili; l'articolo si limita a ciò che è verificabile («copre MCP, LangGraph, RAG e Skills in un unico syllabus MIT»).
